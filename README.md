# Система управления бизнесом

## Описание функционала:

### Управление пользователями

- Регистрация и авторизация по email и паролю
- Три роли пользователей:
    - Обычный сотрудник
    - Менеджер
    - Администратор команды
- Личный кабинет с возможностью обновления профиля
- Удаление аккаунта (без возможности восстановления)

### Организация команд

- Создание компании/команды (доступно администраторам)
- Управление составом команды: добавление и удаление участников
- Назначение ролей (сотрудник/менеджер) для участников команды
- Просмотр полного состава команды

### Система задач

- Создание задач руководителями с назначением исполнителей
- Детализация задач: описание, срок выполнения, статус
- Жизненный цикл задачи:
  Открыта → В работе → Выполнена
- Внутренний чат задачи для обсуждения и комментариев
- Редактирование и удаление задач

### Оценка эффективности

- Оценка выполненных задач руководителем (от 1 до 5 баллов)
- Расчет средней оценки за выбранный период

### Планирование встреч

- Назначение встреч с выбором даты, времени и участников
- Автоматическая проверка на пересечение событий
- Личный календарь встреч каждого сотрудника
- Возможность отмены запланированных встреч

### Интегрированный календарь

- Единое отображение задач и встреч по дням
- Два режима просмотра:
    - Месячный обзор (текстовая таблица)
    - Детальный дневной вид
- Автоматическое добавление дедлайнов задач и встреч в календарь

### Администрирование

- Готовая административная панель
- Управление всеми сущностями:
    - Пользователи и роли
    - Команды и их участники
    - Задачи и их статусы
    - Встречи и расписания

## Для использования:

1. Клонируйте проект:

```
git clone https://github.com/Lykor1/final_project.git
cd final_project
```

2. Создайте файл `.env` в корневой директории проекта на основе примера:
   `cp .env_example .env`
3. Отредактируйте `.env` файл, заполнив необходимые переменные.
4. Соберите и запустите контейнеры Docker:

```
# Запуск в фоновом режиме
docker compose up -d --build

# Запуск с отслеживанием логов
docker compose up --build

# Проверка статуса контейнеров
docker compose ps

# Просмотр логов (всех)
docker compose logs
# Просмотр логов сервиса
docker compose logs <service>
```

5. Примените миграции:
   `docker compose exec web python manage.py migrate`
6. Создайте суперпользователя:
   `docker compose exec web python manage.py createsuperuser`
7. Приступайте к использованию:

```
# Административная панель
http://localhost:8000/admin

# Простой UI для тестирования эндпоинтов
http://localhost:8000/frontend
```
